# Gemini Chatbot

A simple web-based chatbot built using **Google Gemini (Generative AI)** and **Streamlit**. Chat with Gemini models directly from your browser with conversation history.

---

## Features

- Uses **Google Gemini (GenAI)** models for natural language responses.
- Web interface with **Streamlit**.
- Maintains conversation context in session state.
- Simple setup using only an **API key** (no full Google Cloud setup required).

---

## Requirements

- Python 3.9+
- [Streamlit](https://streamlit.io/)
- [Google Generative AI Python SDK](https://pypi.org/project/google-generativeai/)

---

## Installation

1. Clone this repository:
```bash
git clone https://github.com/your-username/gemini-chatbot.git
cd gemini-chatbot
```
2.	Install dependencies:
```bash
pip install streamlit google-generativeai
```
3.	Get your Google API key from AI Studio.

## Usage
1.	Open app.py and replace:
```bash
genai.configure(api_key="YOUR_API_KEY_HERE")
```
with your API key.

2.	Run the chatbot:
```bash
streamlit run example.py
```
3.	A browser window will open. Type messages and chat with Gemini!

## Supported Models

•	The default model is "gemini-2.5-flash"

•	To see other available models:
```bash
import google.generativeai as genai

genai.configure(api_key="YOUR_API_KEY_HERE")
models = genai.list_models()
for m in models:
    print(m.name, m.supported_generation_methods)
```

### How Multi-Turn Conversation Works

This chatbot maintains multi-turn conversation context using the structured `history` passed to the model.

- **Conversation history**: A list of all messages exchanged so far.
- **Role field**: `"user"` for user messages, `"model"` for AI responses. Helps the model distinguish speakers.
- **Parts field**: Contains the actual message content (text, code, etc.).

Example:

```python
history = [
    {"role": "user", "parts": ["Hi, who are you?"]},
    {"role": "model", "parts": ["I'm a chatbot powered by Google Gemini."]}
]
```
Every time model.generate_content(history) is called, the model sees the full history, maintaining context and producing coherent responses across multiple turns.

### How Multi-Turn Conversation Works

This chatbot maintains multi-turn conversation context using the structured `history` passed to the model. This allows the bot to "remember" previous messages and respond coherently.

**Key components:**

- **Conversation History**: A list of all messages exchanged so far between the user and the model.
- **Role Field**: 
  - `"user"` → indicates a message from the user.  
  - `"model"` → indicates a message generated by the AI.  
  - Helps the model distinguish who said what.
- **Parts Field**: A list containing the actual message content (text, code, or other data). Each message can contain multiple parts if needed.

**Example:**

```python
history = [
    {"role": "user", "parts": ["Hi, who are you?"]},
    {"role": "model", "parts": ["I'm a chatbot powered by Google Gemini."]}
]
```
How it works:
	1.	Every new user input is appended to history.
	2.	The model generates a response using the entire history, not just the latest message.
	3.	The model’s response is appended with "role": "model" and added to history.
	4.	This structured approach allows the model to maintain multi-turn context across the conversation.

Benefit:
The bot can produce contextually relevant responses and maintain continuity over multiple turns, instead of treating each message independently.

## Notes
•	Only the roles "user" and "model" are valid.

•	Each message uses parts: [text] instead of content.

•	Conversation context is stored in st.session_state.history.
